Overview

From a flat, broken topology to a hierarchical enterprise network.

This project simulates a small enterprise campus network using Cisco Packet Tracer. It demonstrates foundational network engineering and cybersecurity concepts: VLAN segmentation, 802.1Q trunking, centralized switching architecture, and inter-VLAN routing using Router-on-a-Stick.

The lab began as a flat, unstructured topology that quickly proved unstable and unscalable as departments were added. Without a core switch, trunk and routing conflicts emerged at scale. It was rebuilt from the ground up into a hierarchical enterprise architecture with a dedicated core/distribution switch and multiple departmental access switches — mirroring real-world enterprise infrastructure design patterns.

Network Architecture

Three-tier hierarchical design.

Router0 sits at the top providing inter-VLAN routing via subinterfaces. Switch3 is the core/distribution layer, aggregating every trunk link. Eight access switches fan out to departmental PCs, with the ninth switch hosting the command-and-control private data segment behind the perimeter firewall.

VLAN Segmentation Plan

Logical separation on shared physical infrastructure.

Configuration Highlights

Layer 2 — 802.1Q Trunk

802.1Q tagging stamps each frame with its VLAN ID so the receiving device routes it correctly. Without trunks, you'd need a separate physical cable for every VLAN.

Switch# configure terminal
Switch(config)# interface range fa0/1 - 8
Switch(config-if-range)# switchport mode trunk
Switch(config-if-range)# switchport trunk encapsulation dot1q
Switch(config-if-range)# end

Layer 3 — Router-on-a-Stick

Router-on-a-Stick uses subinterfaces — logical divisions of one physical interface, each tagged with a different VLAN ID. Each subinterface acts as the default gateway for its VLAN.

interface gig0/0
 no shutdown
!
interface gig0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
!
interface gig0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0

VLAN Creation on Switch3

All nine VLANs are created and named on the core/distribution switch, then propagated through trunks to access switches.

Switch(config)# vlan 10
Switch(config-vlan)# name ACCOUNTING
Switch(config-vlan)# vlan 20
Switch(config-vlan)# name HR-FINANCE
Switch(config-vlan)# vlan 30
Switch(config-vlan)# name IT-SECURITY

Troubleshooting Scenarios

Real failures encountered during the build — not simulated.

VID Conflict

Subinterface VID conflict

%Configuration of multiple subinterfaces of the same main interface with the same VID is not permitted.

Root cause: VLAN 10 encapsulation was assigned to the wrong subinterface (gig0/0.20 instead of gig0/0.10).

Fix: Re-entered encapsulation dot1Q 10 on gig0/0.10 and encapsulation dot1Q 20 on gig0/0.20.

Encapsulation Order

IP before encapsulation

% Configuring IP routing on a LAN subinterface is only allowed if that subinterface is already configured as part of an IEEE 802.10, IEEE 802.1Q, or ISL VLAN.

Root cause: ip address was entered before encapsulation dot1Q.

Fix: Always configure encapsulation first, then assign the IP address.

DTP Flap

Trunk negotiation / interface flapping

%LINEPROTO-5-UPDOWN: Interface FastEthernet0/1, changed state to down
%LINEPROTO-5-UPDOWN: Interface FastEthernet0/1, changed state to up

Expected DTP negotiation behavior. All interfaces stabilized to up/up after 802.1Q negotiation completed.

Native VLAN

Native VLAN mismatch

CDP warnings; broadcast leaking across VLANs.

Fix: switchport trunk native vlan must match on both ends.

Topology

Flat network failure → rebuild

The original flat topology connected access switches directly to the router with no core switch, creating trunk and routing conflicts at scale. This forced a full redesign into the current three-tier hierarchical model.

Key takeaway — the most technically valuable part was diagnosing what broke.

The most technically valuable part of this project wasn't writing configs that worked — it was diagnosing why they didn't. Understanding that a subinterface needs encapsulation before accepting an IP, that trunks must match on both ends, that a flat topology breaks at scale — that diagnostic process is what network engineering actually looks like. Rebuilding a broken topology into a working hierarchical enterprise network is the real lab.
